RPC API

HTTP API exposed by every Lattice node. All endpoints return JSON. Default port is 8545.

Sections
Authentication Chain Accounts Blocks Transactions Fee Market DEX (Batch Auction) Light Client Network Observability

Authentication

When the node is started with --rpc-auth, a random 32-byte hex token is written to <dataDir>/.cookie. All requests must include the token:

curl -H "Authorization: Bearer $(cat ~/.lattice/.cookie)" \
  http://localhost:8545/api/chain/info

Without --rpc-auth, all endpoints are open. Use auth in production or when the RPC port is exposed.

Chain

GET /api/chain/info
Chain status: current height, tip block CID, sync state, mining state, mempool size, and peer count.
curl http://localhost:8545/api/chain/info
{
  "height": 142857,
  "tip": "baguqeera...",
  "syncing": false,
  "mining": true,
  "mempool": 23,
  "peers": 8
}
GET /api/chain/spec
Chain specification parameters: block time target, reward schedule, capacity limits, difficulty window.

Accounts

GET /api/balance/{address}
Current balance for the given address.
curl http://localhost:8545/api/balance/baguqeera...abc
{
  "address": "baguqeera...abc",
  "balance": 5242880
}
GET /api/nonce/{address}
Next valid nonce for the given address. Returns stored + 1 for accounts that have transacted, 0 for fresh accounts. Used for transaction construction.
GET /api/proof/{address}
Merkle proof of the account's balance against the current state root. Verifiable by light clients without full state.

Blocks

GET /api/block/latest
Latest block info: height, hash, timestamp, transaction count, difficulty.
GET /api/block/{id}
Block by CID hash or height number. Returns full block data including transactions and child blocks.
# By height
curl http://localhost:8545/api/block/42

# By CID
curl http://localhost:8545/api/block/baguqeera...xyz

Transactions

POST /api/transaction
Submit a signed transaction. The transaction is validated and added to the mempool if valid. Returns the transaction CID.
curl -X POST http://localhost:8545/api/transaction \
  -H "Content-Type: application/json" \
  -d '{
    "signatures": {"<pubkey>": "<signature>"},
    "body": "<bodyCID>"
  }'

Transaction validation rules:

  • Signatures — every signer address must have a valid P256 ECDSA signature over body.rawCID
  • Fee — between 1 and 1,000,000,000,000
  • Nonce — within ±600 blocks of current height (replay protection)
  • Size — serialized body ≤ 102,400 bytes
  • Balance — each oldBalance must match on-chain state
  • Conservation — sum(debits) == sum(credits) + fee
GET /api/receipt/{txCID}
Transaction receipt: block hash, height, timestamp, fee, sender, and balance changes. Recent receipts return instantly; older ones are reconstructed from CAS.
{
  "txCID": "baguqeera...tx",
  "blockHash": "baguqeera...blk",
  "blockHeight": 142850,
  "timestamp": 1742601600000,
  "fee": 100,
  "sender": "baguqeera...abc",
  "accountActions": [
    {"owner": "baguqeera...abc", "oldBalance": 5242880, "newBalance": 4194204},
    {"owner": "baguqeera...def", "oldBalance": 0, "newBalance": 1048576}
  ]
}
GET /api/mempool
Mempool statistics: pending count, total fees, per-account counts. Max 10,000 transactions; max 64 per account.

Fee Market

GET /api/fee/estimate?target=N
Estimated fee for confirmation within N blocks. Higher target = lower fee.
curl http://localhost:8545/api/fee/estimate?target=3
{
  "target": 3,
  "estimatedFee": 150
}
GET /api/fee/histogram
Fee distribution histogram of current mempool transactions. Useful for fee market visualization.

DEX (Batch Auction)

The built-in DEX uses a commit-reveal batch auction to prevent front-running. Orders go through three phases: commit (hash only), reveal (after window), settle (at midpoint price).

GET /api/orders
List active orders in the current auction window.
POST /api/orders
Place a new order directly (no commit-reveal). Suitable for non-MEV-sensitive trades.
POST /api/orders/commit
Commit an order hash for MEV protection. The actual order is hidden until the reveal phase (3-block auction window).
POST /api/orders/reveal
Reveal a previously committed order. Must match the committed hash. Orders are settled at midpoint prices after the auction window.

Light Client

GET /api/light/headers?from=X&to=Y
Block headers for the given height range. Light clients use this for header-chain sync without downloading full blocks.
GET /api/light/proof/{address}
Account proof with chain context: balance proof plus block header at proof height. Sufficient for independent verification.

Network

GET /api/peers
Connected peer list with connection info, reputation scores, and subnet distribution.

Observability

GET /metrics
Prometheus-format metrics. Scrape with any Prometheus-compatible collector.
GET /ws
WebSocket endpoint for real-time block and transaction event subscriptions. Planned — not yet implemented.
