Getting Started

Install, run, mine.

Get a Lattice node running on your machine in under five minutes. Join the network, sync the chain, and start mining blocks.

1 Requirements

  • macOS 14+ or Ubuntu 22.04+
  • Swift 6.0+ — swift.org/install
  • 4 GB RAM minimum (8 GB recommended for mining)
  • 10 GB disk for chain data (grows over time)

Docker users: Skip to the Docker section below if you prefer containers.

2 Install from source

Clone and build

# Clone the node
git clone https://github.com/treehauslabs/lattice-node.git
cd lattice-node

# Build in release mode
swift build -c release

The binary is at .build/release/LatticeNode. Optionally copy it to your PATH:

cp .build/release/LatticeNode /usr/local/bin/lattice-node

3 Run a node

Start on mainnet

lattice-node node --data-dir ~/.lattice

The node will:

  1. Create the data directory if it doesn't exist
  2. Connect to bootstrap peers via hardcoded seeds and DNS
  3. Sync the chain (snapshot sync by default — downloads recent blocks only)
  4. Begin relaying blocks and transactions to the network

Common options

lattice-node node \
  --data-dir ~/.lattice \
  --rpc-port 8545 \
  --p2p-port 30303 \
  --rpc-auth \
  --log-level info
  • --rpc-port — HTTP RPC port (default: 8545)
  • --p2p-port — P2P listen port (default: 30303)
  • --rpc-auth — Enable bearer token auth (writes token to <data-dir>/.cookie)
  • --log-level — debug, info, warning, error

Check sync status

curl http://localhost:8545/api/chain/info | python3 -m json.tool
{
  "height": 142857,
  "tip": "baguqeera...",
  "syncing": false,
  "mining": false,
  "mempool": 23,
  "peers": 8
}

4 Mine

Mining requires a key pair. Generate one first if you haven't already (see the Key Management guide).

# Generate a key pair
lattice-node keys generate --output ~/.lattice/miner.key

# Start mining
lattice-node node \
  --data-dir ~/.lattice \
  --mine \
  --miner-key ~/.lattice/miner.key

The miner uses (CPU cores - 1) parallel workers by default. Each worker searches a disjoint nonce range. When a valid block is found:

  1. Block is stored in local CAS
  2. Pushed to connected peers
  3. Chain state updated and mempool pruned

Block reward: The initial reward is 1,048,576 units per block. Rewards halve every 17,592,186,044,416 blocks. The coinbase transaction credits reward + fees to your miner address.

Merged mining

If the node is configured with child chains, mining a Nexus block automatically produces valid blocks for all child chains. No additional work required — child blocks are embedded in the parent block's childBlocks field.

5 Run a local devnet

For development and testing, spin up an isolated network:

# Single-node devnet with instant mining
lattice-node devnet \
  --nodes 1 \
  --data-dir ~/.lattice-devnet

# Multi-node local cluster
lattice-node cluster \
  --nodes 4 \
  --data-dir ~/.lattice-cluster

Devnet uses lower difficulty so blocks are found immediately. The cluster command starts multiple nodes that discover each other via mDNS.

6 Docker

# Run a node
docker run -d \
  --name lattice \
  -p 8545:8545 \
  -p 30303:30303 \
  -v lattice-data:/data \
  ghcr.io/treehauslabs/lattice-node \
  node --data-dir /data

# Run with mining
docker run -d \
  --name lattice-miner \
  -p 8545:8545 \
  -p 30303:30303 \
  -v lattice-data:/data \
  -v ~/.lattice/miner.key:/keys/miner.key:ro \
  ghcr.io/treehauslabs/lattice-node \
  node --data-dir /data --mine --miner-key /keys/miner.key

Nexus chain parameters

The Nexus genesis chain ships with these defaults:

Target block time 10 seconds
Initial reward 1,048,576
Max transactions/block 5,000
Difficulty adjustment Every 120 blocks (~20 min)
Max block size 10 MB
Genesis March 22, 2025 UTC
