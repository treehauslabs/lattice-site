Wallet

Swift wallet library and reference app for the Lattice network. Key management, transaction signing, cross-chain transfers, and multi-chain balance tracking.

Overview

The Lattice Wallet is both a Swift package you can embed in any app and a reference implementation showing how to manage keys, sign transactions, and perform cross-chain transfers on the Lattice network. It uses Apple's CryptoKit on Apple platforms and swift-crypto on Linux.

Key Management

Generate, import, and export Ed25519 keypairs. Keys are encrypted at rest using AES-256-GCM with a passphrase-derived key.

Transaction Signing

Build and sign transactions locally. Supports single-chain transfers, cross-chain transfers, and state updates.

Multi-Chain Balances

Query balances across all chains in the hierarchy from a single wallet instance. Aggregated and per-chain views.

Cross-Platform

Runs on macOS, iOS, tvOS, watchOS, visionOS, and Linux. Same API everywhere, no platform conditionals needed.

Platform support

macOS iOS tvOS watchOS visionOS Linux

Installation

Swift Package Manager

Add the wallet as a dependency in your Package.swift:

dependencies: [
    .package(
        url: "https://github.com/treehauslabs/lattice-wallet.git",
        from: "0.1.0"
    )
]

Then add "LatticeWallet" to your target's dependencies.

Build from source

# Clone and build
git clone https://github.com/treehauslabs/lattice-wallet.git
cd lattice-wallet
swift build -c release

# Run tests
swift test

Usage

Create a wallet and generate keys

import LatticeWallet

// Create a new wallet with a keystore path
let wallet = try Wallet(
    keystorePath: "~/.lattice/keystore",
    gateway: "http://localhost:8545"
)

// Generate a new keypair
let key = try wallet.createKey(passphrase: "my-secure-passphrase")
print(key.address) // lat1q2w3e4r5t6y7u8...

Check balances

// Balance on a specific chain
let balance = try await wallet.balance(
    address: key.address,
    chain: "payments"
)
print(balance) // 1,250.00

// Balances across all chains
let allBalances = try await wallet.allBalances(address: key.address)
for (chain, amount) in allBalances {
    print("\(chain): \(amount)")
}

Send a transaction

// Single-chain transfer
let tx = try await wallet.send(
    from: key,
    to: "lat1a2b3c4d5e6f7g8...",
    amount: 100,
    chain: "payments",
    passphrase: "my-secure-passphrase"
)
print(tx.hash) // 0xabc123...

// Wait for confirmation
let receipt = try await tx.confirmed()
print(receipt.blockHeight) // 184,294

Cross-chain transfer

// Transfer from payments chain to identity chain
let transfer = try await wallet.crossChainTransfer(
    from: key,
    toAddress: "lat1a2b3c4d5e6f7g8...",
    amount: 50,
    sourceChain: "payments",
    destinationChain: "identity",
    passphrase: "my-secure-passphrase"
)

// The library handles deposit, proof generation, and withdrawal
print(transfer.depositHash)    // 0xabc...
print(transfer.proofSize)      // 96 bytes
print(transfer.withdrawalHash) // 0xdef...

Key Management

Import and export

// Import a raw private key
let imported = try wallet.importKey(
    privateKey: rawKeyBytes,
    passphrase: "passphrase"
)

// Export encrypted keystore (JSON)
let json = try wallet.exportKey(
    address: key.address,
    passphrase: "passphrase"
)

// List all keys in the keystore
let keys = wallet.listKeys()
for k in keys {
    print("\(k.address) — created \(k.createdAt)")
}

Keystore format

Keys are stored as JSON files encrypted with AES-256-GCM. The encryption key is derived from the passphrase using scrypt (N=218, r=8, p=1). Each key file contains:

  • address — The public address derived from the Ed25519 public key
  • crypto — Encrypted private key, cipher params, and KDF params
  • version — Keystore format version (currently 1)

Architecture

The wallet is structured as three layers:

  • LatticeWallet — High-level API: wallet creation, balance queries, transaction sending, cross-chain transfers
  • LatticeSigner — Transaction building and Ed25519 signing. Stateless and testable in isolation.
  • LatticeKeystore — Encrypted key storage. Reads and writes keystore JSON files. No network access.

Each layer can be used independently. If you only need signing, depend on LatticeSigner. If you want to manage keys without network access, use LatticeKeystore.

Security

  • Private keys never leave the keystore unencrypted
  • Signing happens in-memory; key material is zeroed after use
  • No private keys are transmitted over the network
  • The gateway connection is used only for reading state and submitting signed transactions
  • All cryptographic operations use Apple CryptoKit (Apple platforms) or swift-crypto (Linux)
