The Problem

Blockchains don't scale.
Bridges aren't safe.

The industry has tried two approaches. Both have fundamental flaws.

◼

Monolithic Chains

Bitcoin does ~7 TPS. Ethereum L1 does ~15 TPS. One chain processes everything, and every node validates everything. Security is high but throughput has a hard ceiling.

◼

Bridges Are Attack Vectors

Multi-chain ecosystems like Cosmos and Avalanche scale throughput but rely on trusted bridges for cross-chain transfers. Over $2 billion stolen in bridge exploits since 2022.

◆

Lattice: A Third Way

A tree of chains where parents secure children through merged mining. Cross-chain transfers verified by Merkle proofs against committed state roots. No middlemen. Just math.

Architecture

A blockchain that grows like a tree

Every parent chain secures its children. Every child inherits the full hashrate of its parent through nested merged mining.

Nexus Chain ← root, highest security, processes cross-chain receipts | |--- Payment Chain ← high-throughput value transfer | |--- Region-US | |--- Region-EU | \--- Region-APAC | |--- Data Chain ← content-addressed storage proofs | |--- Media | \--- Documents | \--- Identity Chain ← decentralized identity & credentials |--- Personal \--- Enterprise Each chain mines simultaneously with one nonce search. N sibling chains = N× aggregate throughput, same security.
How It Works

Three innovations, one protocol

01

Nested Merged Mining

Every parent block embeds child block headers. A single proof-of-work search produces valid blocks for the entire tree simultaneously. Child chains inherit the full hashrate of the nexus chain — no security fragmentation.

02

Trustless Cross-Chain

No bridges. No oracles. No multisigs. Every child block commits a snapshot of its parent's state root. Cross-chain transfers are verified via Merkle proofs against these committed roots. The proof is the protocol.

03

Content-Addressed Everything

Blocks, transactions, and state are addressed by their cryptographic hash. Nodes only download data they care about. Light clients verify Merkle proofs without full state. If the hash matches, the content is correct.

Cross-Chain Transfers

Three steps, zero trust assumptions

Deposit, receipt, withdrawal — each verified by Merkle proof against committed state roots.

1. Deposit

Quick Start

Five lines to a running node

  main.swift 
import Acorn
import AcornMemoryWorker
import AcornDiskWorker
import Ivy

// Build the storage chain: memory → disk → network
let memory = MemoryCASWorker()
let disk = DiskCASWorker(directory: "/data/blocks")

// Start a P2P node with trust-line routing
let node = Ivy(config: IvyConfig(
    publicKey: myKey,
    bootstrapPeers: [seed1, seed2],
))
try await node.start()

// Fetch any content by hash — cascades through the chain
let data = await chain.get(cid: blockHash)
