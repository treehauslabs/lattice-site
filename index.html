How everyone else does it

Independent chains with separate validator sets. Moving value between chains requires a trusted intermediary. That intermediary is a single point of failure holding billions in locked funds. One compromised key, and it's gone.

How Lattice does it

A tree of chains where every child block commits its parent's state root. Cross-chain transfers are verified by Merkle proof against committed roots. No multisig. No relayer. No oracle. The cryptographic proof IS the bridge.

Other multi-chain: add chains, divide security.
Lattice: add chains, multiply throughput. Security stays whole.

Every parent block embeds child block headers. One hash search produces valid blocks for the entire tree simultaneously. Child chains don't compete for hashrate — they share it. Adding a hundred chains costs zero additional security budget.

How It Works

Three innovations, one protocol

◆

Nested Merged Mining

One hash, every chain.

Every parent block embeds child headers. A single PoW search produces valid blocks across the entire hierarchy. Child chains inherit the full hashrate of the root — no fragmentation, no additional energy cost.

◆

Trustless Cross-Chain

The proof is the bridge.

Every child block snapshots its parent's state root. Transfers are verified via Merkle proof against committed roots. No relayers. No oracles. No multisig wallets. Deposit, receipt, withdrawal — each step cryptographically proven.

◆

Content-Addressed State

If the hash matches, it's correct.

Every block, transaction, and state tree node is addressed by its SHA-256 hash. Nodes fetch only the data they need. Light clients verify Merkle proofs without full state. Deduplication is automatic and free.
