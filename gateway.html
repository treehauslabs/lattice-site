Gateway

HTTP and JSON-RPC gateway to the Lattice network. Query blocks, submit transactions, and stream chain events over WebSocket.

Overview

The Lattice Gateway exposes the node's internal APIs over HTTP, JSON-RPC, and WebSocket. It is the primary interface for wallets, explorers, and applications to interact with the Lattice network. Built in Swift using the same actor-based concurrency model as the rest of the stack.

REST + JSON-RPC

Query chain state, blocks, and transactions via familiar HTTP endpoints or JSON-RPC 2.0 calls.

WebSocket Streams

Subscribe to new blocks, pending transactions, and cross-chain events in real time.

Multi-Chain Aware

Target any chain in the hierarchy by ID. Query cross-chain proofs and state roots across siblings.

Lightweight

Runs as a standalone process or embedded in a full node. Configurable rate limiting and CORS.

Quick Start

Build and run

# Clone and build
git clone https://github.com/treehauslabs/lattice-gateway.git
cd lattice-gateway
swift build -c release

# Run with defaults (localhost:8545)
.build/release/lattice-gateway

# Custom port and node endpoint
.build/release/lattice-gateway --port 8080 --node 127.0.0.1:9000

Configuration

The gateway reads from gateway.toml or command-line flags. Key options:

  • --port — HTTP listen port (default: 8545)
  • --node — Lattice node RPC address to connect to
  • --ws — Enable WebSocket endpoint (default: enabled)
  • --cors — Allowed origins for CORS (default: *)
  • --rate-limit — Max requests per second per IP (default: 100)
  • --chain — Default chain ID when none is specified in requests

REST Endpoints

GET /v1/blocks/:hash Fetch block by hash
GET /v1/blocks/latest Latest block on chain
GET /v1/blocks/:height Fetch block by height
GET /v1/tx/:hash Transaction by hash
POST /v1/tx Submit signed transaction
GET /v1/chain/:id/state/:key Read state from a specific chain
GET /v1/proof/:chain/:key Merkle proof for cross-chain verification
WS /v1/ws WebSocket stream

JSON-RPC

The gateway also accepts JSON-RPC 2.0 calls on the same port. Send a POST request with Content-Type: application/json. 

// Request
{
  "jsonrpc": "2.0",
  "method": "lattice_getBlock",
  "params": { "height": 42 },
  "id": 1
}

// Response
{
  "jsonrpc": "2.0",
  "result": {
    "hash": "a1b2c3...",
    "height": 42,
    "parent": "d4e5f6...",
    "stateRoot": "789abc...",
    "txCount": 17
  },
  "id": 1
}

Available methods

  • lattice_getBlock — Get block by height or hash
  • lattice_getTransaction — Get transaction by hash
  • lattice_sendTransaction — Submit a signed transaction
  • lattice_getState — Read state by key on a given chain
  • lattice_getProof — Merkle proof for a state key
  • lattice_chainInfo — Chain metadata (height, hash rate, peer count)
  • lattice_subscribe — Subscribe to events (WebSocket only)

WebSocket Subscriptions

Connect to /v1/ws and send a lattice_subscribe call to start receiving events. 

// Subscribe to new blocks on chain "payments"
{
  "jsonrpc": "2.0",
  "method": "lattice_subscribe",
  "params": {
    "event": "newBlock",
    "chain": "payments"
  },
  "id": 1
}

Supported events:

  • newBlock — Emitted when a new block is committed
  • pendingTx — Emitted when a transaction enters the mempool
  • crossChain — Emitted on cross-chain transfer completions
  • stateChange — Emitted when a specific state key is modified
